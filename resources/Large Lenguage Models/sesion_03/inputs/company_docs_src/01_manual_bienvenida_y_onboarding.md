# 1. Objetivo

Este manual describe el proceso estándar de incorporación de una persona a NovaCorp Iberia. Su finalidad es que cada empleado conozca qué ocurre antes de su primer día, qué accesos recibirá, qué reuniones son obligatorias y cómo resolver incidencias habituales durante sus primeras semanas.

El documento no sustituye al contrato laboral, al convenio aplicable ni a las políticas específicas de seguridad, IT o protección de datos. Cuando exista discrepancia, prevalecerá la política específica publicada en el portal corporativo.

# 2. Resumen ejecutivo del proceso

La incorporación se organiza en cuatro fases:

1. **Pre-alta**: recopilación de documentación, creación del expediente y solicitud de equipo.
2. **Primer día**: activación de usuario, entrega de equipo, bienvenida con manager y sesión de cultura.
3. **Primeras dos semanas**: formación obligatoria, lectura de políticas y planificación del puesto.
4. **Primeros noventa días**: seguimiento de objetivos, feedback estructurado y cierre del onboarding.

Cada fase tiene responsables definidos para evitar ambigüedades: Personas coordina el alta administrativa, IT habilita las herramientas, el manager define prioridades y el buddy acompaña la integración informal.

# 3. Antes del primer día

## 3.1 Documentación necesaria

La persona recibirá un enlace seguro para aportar la documentación de alta. Debe completar el formulario antes de las 12:00 del tercer día laborable anterior a la incorporación. Los documentos habituales son:

- documento de identidad o NIE;
- número de afiliación a la Seguridad Social;
- datos bancarios para nómina;
- certificado de retenciones si procede;
- contacto de emergencia;
- aceptación de políticas internas.

Si falta documentación, Personas abrirá una tarea en el portal de empleados. No debe enviarse documentación sensible por correo ordinario salvo instrucción expresa de Personas.

## 3.2 Preparación del equipo

IT prepara portátil, cargador, auriculares y credenciales iniciales. En incorporaciones remotas, el envío se programa con al menos dos días laborables de margen. En incorporaciones presenciales, el equipo se entrega en recepción tras validar la identidad.

El manager debe confirmar si la persona necesita licencias adicionales: CRM, herramientas analíticas, repositorios técnicos, entornos de preventa o acceso a documentación de cliente.

# 4. Primer día

## 4.1 Agenda recomendada

| Hora | Actividad | Responsable |
|---|---|---|
| 09:30 | Bienvenida y comprobación de equipo | Buddy |
| 10:00 | Activación de cuenta, MFA y VPN | IT |
| 11:30 | Reunión con manager | Manager directo |
| 12:30 | Presentación del equipo | Manager |
| 15:00 | Sesión de cultura, valores y normas internas | Personas |
| 16:00 | Lectura guiada de políticas críticas | Empleado |

La agenda puede variar por zona horaria o disponibilidad del equipo, pero la activación de cuenta y MFA debe realizarse el primer día.

## 4.2 Accesos iniciales

La cuenta corporativa permite acceder al correo, portal de empleados, intranet, sistema de RRHH, gestor de tickets y repositorio documental. La contraseña inicial caduca al final del primer día y debe cambiarse durante la activación.

El portal de empleados está disponible en `https://portal.novacorp.example`. Para acceder desde fuera de la red corporativa se requiere VPN activa y autenticación multifactor.

# 5. Formación obligatoria

Durante las dos primeras semanas deben completarse las siguientes formaciones:

- seguridad de la información;
- protección de datos y confidencialidad;
- prevención de riesgos laborales;
- código ético y canal de denuncias;
- uso aceptable de herramientas corporativas;
- procesos básicos de gastos, compras y vacaciones.

El incumplimiento de estas formaciones genera recordatorios automáticos. Si una formación sigue pendiente tras quince días naturales, el manager recibe una alerta.

# 6. Roles de acompañamiento

## 6.1 Manager directo

Define el contexto del puesto, prioridades, métricas de éxito y relación con otros equipos. Debe agendar una reunión semanal durante el primer mes.

## 6.2 Buddy

Ayuda con dudas operativas: dónde encontrar documentación, cómo se organizan las reuniones, qué canales de comunicación se usan y cómo escalar incidencias internas.

## 6.3 Personas

Resuelve dudas de contrato, horario, beneficios, vacaciones, ausencias, desempeño y nómina.

# 7. Primeros 90 días

Al finalizar el primer mes, el manager registra una revisión breve con tres elementos: adaptación al puesto, bloqueos principales y próximos objetivos. Al finalizar los 90 días, se realiza una revisión formal de onboarding. No es una evaluación anual de desempeño, sino una comprobación de integración y expectativas.

# 8. Incidencias frecuentes

| Situación | Qué hacer |
|---|---|
| No llega el equipo | Abrir ticket en Soporte IT con copia al buddy |
| No funciona el MFA | Usar el flujo de recuperación de identidad y avisar a IT |
| No aparece una formación | Consultar en Portal de empleados > Formación |
| Faltan accesos a una herramienta | Pedir al manager que apruebe la solicitud en el catálogo de accesos |
| Dudas sobre contrato | Abrir consulta en Portal de empleados > Personas |

# 9. Criterios de escalado

Se debe escalar a Personas si la incidencia afecta a contrato, salario, jornada o permisos. Se debe escalar a IT si impide trabajar más de cuatro horas. Se debe escalar a Seguridad si se detecta pérdida de equipo, acceso indebido o exposición de información confidencial.
