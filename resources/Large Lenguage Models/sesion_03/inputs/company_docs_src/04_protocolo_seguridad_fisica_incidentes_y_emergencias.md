# 1. Objetivo

Este protocolo establece cómo prevenir, reportar y gestionar incidentes de seguridad física en oficinas y actividades corporativas. Incluye accesos a edificios, visitas, emergencias, pérdida de equipos y situaciones que puedan afectar a personas, instalaciones o información.

# 2. Acceso a oficinas

El acceso a oficinas se realiza con tarjeta corporativa personal. No se permite prestar tarjetas ni permitir el paso a personas no identificadas. Las visitas deben registrarse previamente y permanecer acompañadas por un empleado anfitrión.

Si una tarjeta se pierde, se debe reportar de inmediato para bloquearla. Recepción puede emitir una tarjeta temporal de un día tras validar identidad.

# 3. Reporte de incidentes

## 3.1 Qué se considera incidente

Se considera incidente cualquier evento que pueda afectar a personas, instalaciones, continuidad operativa o confidencialidad. Ejemplos:

- persona no autorizada en zona restringida;
- robo o pérdida de portátil, móvil o tarjeta;
- puerta de emergencia bloqueada;
- amenaza, agresión o comportamiento intimidatorio;
- incendio, inundación, fuga o daño estructural;
- visualización accidental de documentación confidencial por terceros.

## 3.2 Cómo reportar un problema de seguridad

El canal ordinario es **Intranet > Seguridad > Reportar incidente**. El reporte debe incluir:

1. fecha y hora;
2. ubicación exacta;
3. descripción de lo ocurrido;
4. personas afectadas o testigos;
5. fotografías si no comprometen la seguridad;
6. medidas inmediatas tomadas.

Si existe riesgo para personas, se debe llamar primero a emergencias locales y avisar a recepción o seguridad del edificio. El reporte interno se completa después.

# 4. Clasificación de severidad

| Severidad | Descripción | Ejemplos | Tiempo objetivo |
|---|---|---|---|
| Crítica | Riesgo inmediato para personas o instalaciones | incendio, agresión, evacuación | inmediato |
| Alta | Riesgo para activos o información sensible | robo de portátil, intrusión | 1 hora |
| Media | Incidencia controlada sin daño inmediato | puerta averiada, visita no registrada | 1 día laborable |
| Baja | Observación preventiva | mejora de señalización | 5 días laborables |

# 5. Pérdida o robo de equipo

La pérdida de un equipo corporativo se reporta a Seguridad e IT. Seguridad registra el incidente y coordina acciones físicas; IT bloquea credenciales, revoca sesiones y ejecuta borrado remoto si procede.

El empleado debe indicar última ubicación conocida, hora aproximada, si el equipo estaba bloqueado y si contenía documentación sensible.

# 6. Emergencias y evacuación

Cada sede cuenta con planos de evacuación visibles. Durante una alarma, se debe abandonar el puesto sin recoger objetos personales si hacerlo retrasa la salida. El punto de encuentro figura en la señalización de cada edificio.

Los responsables de evacuación verifican zonas asignadas, pero no deben ponerse en riesgo. Nadie debe volver al edificio hasta autorización de seguridad o servicios de emergencia.

# 7. Protección de información en oficinas

Se aplica política de mesa limpia. Documentos impresos con información interna o confidencial deben guardarse bajo llave o destruirse en contenedores seguros. Las pantallas deben bloquearse al ausentarse del puesto.

Las salas de reunión deben quedar limpias tras una sesión con clientes, proveedores o información sensible. Las pizarras se borran al finalizar.

# 8. Criterios de escalado

Se escala inmediatamente a Seguridad Corporativa cualquier incidente crítico o alto. Se escala a IT si afecta a dispositivos, credenciales o datos digitales. Se escala a Personas si afecta a integridad, acoso, conflicto laboral o bienestar de empleados.
